## 1. 硬件清单与规格

### 1.1 必选硬件

| 序号 | 物品 | 规格参数 | 数量 | 备注 |
|------|------|----------|------|------|
| 1 | 树莓派 | Raspberry Pi 3B/3B+/4B/5 | 1 | 推荐 4B 或 5，需 40Pin GPIO |
| 2 | NeoPixel 矩阵 | WS2812B，逻辑 24×7，5V，GRB | 1 | 物理 LED 数 = 168 × LEDS_PER_GRID（默认 2 → 336 颗） |
| 3 | microSD 卡 | 32GB+，Class 10 | 1 | 用于树莓派系统 |
| 4 | 树莓派电源 | 5V 3A（Type-C 或 microUSB） | 1 | 仅给树莓派供电 |
| 5 | NeoPixel 电源 | 5V 3A～5A | 1 | 单独为 LED 供电 |
| 6 | 杜邦线 | 公对母，20cm 左右 | 3 根 | DIN、GND、可选 VCC |
| 7 | 电源线 | 匹配 NeoPixel 接口（如 DC 母头） | 1 根 | 连接外接电源到矩阵 |

### 1.2 可选硬件

| 物品 | 用途 |
|------|------|
| PS4 手柄 | 游戏与模式切换（USB 或蓝牙） |
| 330Ω～470Ω 电阻 | DIN 串联，增强信号稳定性 |
| 74HCT245 或 74AHCT125 | 3.3V→5V 电平转换（信号异常时使用） |
| 1000µF 电容 | 并接在 NeoPixel 电源端，滤波 |
| 保险丝 | 5V 电源正极串联，过流保护 |

### 2.1 引脚图（40Pin 排针俯视）

```
                    3.3V  [ 1] [ 2]  5V
                   GPIO2  [ 3] [ 4]  5V
                   GPIO3  [ 5] [ 6]  GND
                   GPIO4  [ 7] [ 8]  GPIO14
                     GND  [ 9] [10]  GPIO15
                  GPIO17  [11] [12]  GPIO18  ← 数据线接此处
                  GPIO27  [13] [14]  GND
                  GPIO22  [15] [16]  GPIO23
                    3.3V  [17] [18]  GPIO24
                  GPIO10  [19] [20]  GND
                  GPIO9   [21] [22]  GPIO25
                  GPIO11  [23] [24]  GPIO8
                     GND  [25] [26]  GPIO7
                   ...    ...  ...
```

### 2.2 本项目使用的引脚

| 引脚 | BCM 编号 | 物理引脚 | 用途 |
|------|----------|----------|------|
| GPIO 18 | 18 | 12 | NeoPixel 数据线 DIN |
| GND | GND | 6、9、14、20、25、30、34、39 | 共地（任选其一） |

> 若需更改数据引脚，编辑 `drivers.py` 中的 `PIXEL_PIN = board.D18`，可改为 `board.D12`、`board.D21` 等。

## 3. NeoPixel 矩阵详解

### 3.1 兼容型号

| 型号 | 兼容性 | 说明 |
|------|--------|------|
| WS2812B | ✅ 完全兼容 | 默认支持，GRB 顺序 |
| WS2811 | ✅ 通常兼容 | 多为 GRB |
| SK6812 | ⚠️ 可能需改 ORDER | 部分为 RGBW，需查规格 |
| APA102 | ❌ 不兼容 | 双线协议，需更换驱动库 |

### 3.2 接口定义

常见 NeoPixel 矩阵板载接口（以 4Pin 为例）：

```
    ┌─────────────────────┐
    │  [VCC] [GND] [DIN] [DOUT]  │  ← 面向 PCB 正面
    │   5V    地   数据入  数据出  │
    └─────────────────────┘
```

| 标号 | 全称 | 说明 | 接法 |
|------|------|------|------|
| VCC | 电源正极 | 5V 供电 | 接 5V 电源正极 |
| GND | 地 | 电源负极 | 与树莓派 GND、电源 GND 共地 |
| DIN | Data In | 数据输入 | 接 GPIO 18 |
| DOUT | Data Out | 数据输出 | 级联下一块板时用，本项目不接 |

部分板子标为 **+5V / - / IN** 或 **5V / GND / DATA**，含义相同。

### 3.3 尺寸与排布

- **逻辑分辨率**：24 列 × 7 行 = 168 个格子
- **物理 LED 数量**：`168 × LEDS_PER_GRID`（见 `drivers.py`，默认 2 → 336 颗）
- **排布方式**：蛇形（Z 字形）
  - 第 0、2、4、6 行：从左到右（x: 0→23）
  - 第 1、3、5 行：从右到左（x: 23→0）

若你的矩阵尺寸或排布不同，必须修改 `drivers.py` 中的 `set_pixel` 索引计算逻辑。

## 4. 接线步骤

### 4.1 断电操作

所有接线必须在 **树莓派和电源均断电** 的情况下完成，以免短路损坏设备。

### 4.2 接线顺序

**第一步：共地**

1. 将 NeoPixel 的 GND 与树莓派任一 GND 引脚相连（如物理引脚 6）。
2. 若使用独立 5V 电源，将其 GND 也与树莓派 GND 相连。
3. 三者（树莓派 GND、NeoPixel GND、5V 电源 GND）必须共地。

**第二步：数据线**

1. 杜邦线一端接树莓派 **物理引脚 12**（GPIO 18）。
2. 另一端接 NeoPixel 的 **DIN**（或 DATA/IN）。
3. 建议在数据线上串联 330Ω 电阻（靠近树莓派端），减少反射。

**第三步：供电**

1. NeoPixel 的 VCC 接 5V 电源正极。
2. 5V 电源负极接 NeoPixel GND（已与树莓派共地）。
3. 树莓派使用自身电源供电，不要用 LED 电源给树莓派供电。

### 4.3 接线示意

```
                    ┌──────────────────┐
                    │    树莓派        │
                    │                  │
     NeoPixel DIN ──┤ GPIO18 (Pin 12)  │
     NeoPixel GND ──┤ GND (Pin 6)      │
                    └──────────────────┘

                    ┌──────────────────┐
                    │  5V 独立电源     │
                    │  (3A～5A)        │
     NeoPixel VCC ──┤ 正极 (+)         │
     NeoPixel GND ──┤ 负极 (-) ──┬─────┘
                    └────────────│──────
                                 │
                                 └────── 与树莓派 GND 相连（共地）
```

### 4.4 线材与插接

- 数据线尽量短于 30cm，过长易受干扰。
- 杜邦线需插到底，避免虚接。
- 电源线径建议 ≥0.5mm²，大电流时用更粗线缆。

## 5. 供电方案

### 5.1 电流估算

物理 LED 总数取决于 `LEDS_PER_GRID`。

- `LEDS_PER_GRID = 1` → 168 颗
- `LEDS_PER_GRID = 2` → 336 颗（当前默认）

| 场景 | 单颗 LED 电流 | 168 颗最大电流 | 336 颗最大电流 |
|------|---------------|----------------|----------------|
| 全白最亮 | ~60mA | ~10A | ~20A |
| 本项目 brightness=0.4 | ~25mA | ~4.2A | ~8.4A |
| 时钟/游戏典型 | ~10mA | ~1.7A | ~3.4A |

建议 NeoPixel 电源容量：

- `LEDS_PER_GRID = 1`（168 颗）：**5V 3A 起步**
- `LEDS_PER_GRID = 2`（336 颗）：建议 **5V 5A 起步**（更稳妥可选 5V 8A～10A，取决于亮度与同时点亮比例）

### 5.2 方案一：独立 5V 电源（推荐）

- 树莓派：原装 5V 3A Type-C / microUSB 供电。
- NeoPixel：5V 3A～5A 开关电源供电。
- 两者 GND 必须连接在一起。

优点：稳定、互不干扰，适合长时间运行。

### 5.3 方案二：树莓派供电（仅测试用）

- 仅用树莓派 5V（Pin 2/4）和 GND 给 NeoPixel 供电。
- 树莓派 5V 输出能力有限，全亮易掉电重启。
- 仅建议做简单点亮测试，正式使用请用独立电源。

### 5.4 电源接口

常见 NeoPixel 板载接口：

- DC 圆孔（5.5×2.1mm 或 5.5×2.5mm）
- 接线端子（螺丝固定 VCC/GND）
- 焊盘（需自行焊接）

选购电源时注意接口匹配，或另购转接线。

## 6. 电平转换与保护电路（可选）

### 6.1 为何可能需要

- 树莓派 GPIO 输出 3.3V 高电平。
- 部分 NeoPixel 在 3.3V 下工作不稳定，出现闪烁、颜色错乱。
- 通过电平转换将 3.3V 转为 5V 可提高稳定性。

### 6.2 串联电阻

在 GPIO 18 与 DIN 之间串联 **330Ω～470Ω** 电阻：

```
树莓派 GPIO18 ──[330Ω]── NeoPixel DIN
```

可削弱信号过冲，改善长线传输。

### 6.3 74HCT245 电平转换

当 3.3V 驱动不稳定时，可使用 5V 电平：

```
树莓派 GPIO18 ── 74HCT245 A 端
74HCT245 B 端 ── NeoPixel DIN
74HCT245 VCC ── 5V（与 NeoPixel 共电源）
74HCT245 GND ── GND
```

具体引脚以芯片手册为准。

### 6.4 电源滤波

在 NeoPixel 的 VCC 与 GND 之间并接 **100～1000µF** 电解电容（注意正负极），可减小电压波动对显示的影响。

## 7. LED 排布与索引映射

### 7.1 蛇形排布示意

```
行 y=0:  [0]  [1]  [2]  ... [23]   ← 从左到右（逻辑索引）
行 y=1:  [47] [46] [45] ... [24]   ← 从右到左（逻辑索引）
行 y=2:  [48] [49] ... [71]        ← 从左到右（逻辑索引）
...
行 y=6:  [141][142]...[167]      ← 从左到右（逻辑索引）
```

### 7.2 坐标到索引公式

`drivers.py` 中的映射逻辑（先得到“逻辑索引”）：

- 偶数行 (y=0,2,4,6)：`index = y * 24 + x`
- 奇数行 (y=1,3,5)：`index = y * 24 + (23 - x)`

当 `LEDS_PER_GRID > 1` 时，会把逻辑索引映射到一段连续的物理 LED：

- `base = logical_index * LEDS_PER_GRID`
- 该格子的物理 LED 索引为：`base ... base + LEDS_PER_GRID - 1`

若你的矩阵排布不同（如全部从左到右、或不同宽高），需据此修改 `set_pixel` 函数。

### 7.3 每个格子包含多颗 LED（LEDS_PER_GRID）

本项目支持「一个逻辑格子」对应多颗物理 LED，通过 `drivers.py` 的 `LEDS_PER_GRID` 配置实现：

- **逻辑格子数**：固定 24×7 = 168 格
- **物理 LED 总数**：`168 × LEDS_PER_GRID`
- **物理索引映射**：每个逻辑格子占用连续 `LEDS_PER_GRID` 个物理 LED  
  例如 `LEDS_PER_GRID = 2` 时：逻辑索引为 10 的格子 → 物理索引 `20` 和 `21`

#### 布线/排布必须满足的条件

为保证显示不“错位/斑马纹”，你的硬件布线需满足：

- **同一格子的多颗 LED 在物理序列中必须连续**（如 A、B 紧挨着）
- **整墙按蛇形（Z 字形）逐行连接**（与 7.2 的偶数行/奇数行方向一致）

如果你的硬件是“每格两灯但不连续 / 不是蛇形 / 走线跨行”，需要改 `set_pixel()` 的映射逻辑。

### 7.4 不同尺寸的修改

例如 16×8 矩阵：

```python
# drivers.py
WIDTH = 16
HEIGHT = 8
LEDS_PER_GRID = 2  # 每格物理 LED 数量（按需调整）
NUM_PIXELS = WIDTH * HEIGHT * LEDS_PER_GRID
```

并同步修改 `clock_app.py`、`snake_app.py` 中的坐标范围与布局。

## 8. 可选外设连接

### 8.1 PS4 手柄

**USB 连接**

- 用 Micro-USB 线连接手柄与树莓派 USB 口。
- 系统通常自动识别为 `/dev/input/js0`。

**蓝牙连接**

- 树莓派自带蓝牙，安装 `bluez`、`pi-bluetooth`。
- 使用 `bluetoothctl` 配对，手柄以 HID 模式工作。

### 8.2 其他设备

若增加传感器、按钮等，注意：

- 使用 3.3V 兼容的模块。
- 避免与 GPIO 18 冲突。
- 大电流设备勿直接接 GPIO，需通过继电器或驱动电路。

## 9. 常见接线错误与排查

### 9.1 错误接法

| 错误 | 后果 | 纠正 |
|------|------|------|
| GND 未共地 | 信号混乱、不亮或乱码 | 树莓派 GND、NeoPixel GND、电源 GND 连在一起 |
| VCC 接 3.3V | LED 不亮或极暗 | 改为 5V |
| DIN 与 DOUT 接反 | 无显示 | DIN 接树莓派，DOUT 不接或接下级 DIN |
| 电源正负极反接 | 可能烧坏 LED | 确认 VCC 接 +，GND 接 - |
| 树莓派 5V 与 NeoPixel 混用大电流 | 树莓派掉电 | NeoPixel 用独立 5V 电源 |

### 9.2 显示异常排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 完全不亮 | 供电、共地、DIN 接错 | 检查接线，用万用表测电压 |
| 闪烁、花屏 | 3.3V 驱动不稳、线太长 | 加电阻、缩短线、考虑电平转换 |
| 颜色错误 | RGB/GRB 顺序不对 | 修改 `drivers.py` 中 `ORDER` |
| 部分不亮 | 某颗 LED 损坏或虚接 | 逐段排查，替换故障 LED |
| 第一颗亮后面不亮 | 数据线松脱或断线 | 检查 DIN 连接，重插 |

### 9.3 安全提示

- 接线前务必断电。
- 避免电源短路（VCC 与 GND 相触）。
- 大功率电源注意散热与通风。
- 长期运行建议加保险丝或限流保护。

## 附录

### A. 无硬件模拟测试

在无树莓派和 NeoPixel 的情况下，可使用模拟模式进行软件测试：

- 安装 `pip install -r requirements-mock.txt`（仅需 Flask）
- 运行 `python main.py`，程序会进入模拟模式
- 浏览器会自动打开，页面顶部像素预览与游戏逻辑同步显示
- 详见 [DEPLOYMENT.md](DEPLOYMENT.md) 第 11 节

### B. 快速检查清单

接线完成后可逐项确认：

- [ ] 树莓派 GND 与 NeoPixel GND 已连接
- [ ] NeoPixel 使用 5V 供电（独立电源或树莓派 5V）
- [ ] DIN 接树莓派 GPIO 18（物理引脚 12）
- [ ] 电源极性正确（VCC 接 +，GND 接 -）
- [ ] 若用独立电源，其 GND 已与树莓派共地
- [ ] 所有接头插紧、无松动
- [ ] 上电前再次检查无误
