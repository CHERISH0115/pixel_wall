================================================================================
                    Pixel Wall 像素墙 - 项目文件说明
================================================================================

本文档说明该项目中各个文件的功能与作用。

--------------------------------------------------------------------------------
一、程序入口与主控
--------------------------------------------------------------------------------

main.py
    项目主入口文件。
    - 启动 Flask Web 服务器（端口 5000），提供遥控界面与像素预览 API
    - 启动 PS4 手柄监听线程（Linux 下可用）
    - 运行主循环，处理 Web/手柄输入指令
    - 在时钟模式与贪吃蛇模式间切换
    - 无硬件时自动进入模拟模式，可自动打开浏览器
    - 模拟模式下默认进入贪吃蛇，便于测试

--------------------------------------------------------------------------------
二、应用逻辑
--------------------------------------------------------------------------------

clock_app.py
    时钟显示模块。
    - 定义 0～9 和冒号的 5×3 点阵字形
    - 从系统获取当前时间（HH:MM:SS）
    - 在 LED 矩阵上绘制时、分、秒（青色）和冒号（黄色）
    - 每秒刷新一次显示

snake_app.py
    贪吃蛇游戏模块。
    - 实现蛇的移动、转向、碰撞检测
    - 随机生成食物，吃食物后蛇身变长、得分增加
    - 控制蛇头（亮绿）、蛇身（绿色）、食物（红色）的显示
    - 死亡时全屏红色，支持重置重新开始
    - 按键仅更新方向，移动由定时器统一驱动（避免重复移动）

--------------------------------------------------------------------------------
三、硬件驱动
--------------------------------------------------------------------------------

drivers.py
    NeoPixel LED 矩阵驱动层。
    - 配置 24×7 矩阵、GPIO 18、GRB 色序
    - 支持每个逻辑格子对应多颗物理 LED：LEDS_PER_GRID（默认 2 → 336 颗物理 LED）
      - 要求：同一格子的多颗 LED 在物理序列中必须连续，并按蛇形逐行连接
    - 维护 display_buffer 软件缓冲区，供 Web 预览同步
    - 提供 set_pixel(x, y, color)、fill(color)、refresh()、clear()
    - 无硬件时自动进入模拟模式，LED 不输出但预览可用

--------------------------------------------------------------------------------
四、Web 界面
--------------------------------------------------------------------------------

templates/index.html
    Web 遥控器与像素预览页面。
    - 像素预览：Canvas 绘制 24×7 网格，每 100ms 同步 /api/display
    - 连接状态：显示「已连接」或「无法连接」
    - 方向键按钮（▲▼◀▶），控制贪吃蛇方向
    - 切换模式、重置游戏按钮
    - 支持触摸、鼠标、键盘方向键
    - 移动端支持震动反馈

--------------------------------------------------------------------------------
五、API 接口
--------------------------------------------------------------------------------

GET /api/display
    返回当前 24×7 像素矩阵的 RGB 数据（JSON 数组），供浏览器预览同步。

POST /control
    接收遥控指令：{ "btn": "UP"|"DOWN"|"LEFT"|"RIGHT"|"SWITCH_MODE"|"RESET" }

--------------------------------------------------------------------------------
六、依赖与文档
--------------------------------------------------------------------------------

requirements.txt
    树莓派完整依赖（含平台条件，Windows/Mac 仅安装 Flask）。
    - flask、pyPS4Controller、adafruit-circuitpython-neopixel、
      adafruit-blinka、RPi.GPIO

requirements-mock.txt
    无树莓派测试用依赖，仅 flask。
    使用：pip install -r requirements-mock.txt

run.bat
    Windows 下双击运行脚本。自动激活 .venv（若存在）并启动 main.py，
    出错时保持窗口打开。

README.md
    项目总览、快速开始、文档索引。

DEPLOYMENT.md
    从零部署教程：硬件清单、NeoPixel 接线、树莓派系统、依赖安装、
    项目部署、运行、Web 访问、PS4 手柄、开机自启、模拟模式测试、常见问题。

hardware.md
    硬件连接详细指南：GPIO 引脚、NeoPixel 接口、接线步骤、
    供电方案、电平转换、LED 排布、排错方法。

readme.txt
    本文件，项目各文件功能说明。
